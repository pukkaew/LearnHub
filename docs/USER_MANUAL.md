# บทที่ 1: แนะนำระบบ LearnHub

## 1.1 ภาพรวมระบบ

```

├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐ │
│   │  📚 หลักสูตร │    │  📝 ข้อสอบ  │    │  📰 บทความ  │    │  🏆 รางวัล  │ │
│   │   Courses   │    │    Tests    │    │  Articles   │    │   Badges    │ │

│          │                  │                  │                  │        │

│                             │                  │                           │

│                     │         👥 ผู้ใช้งาน              │                   │

## 1.2 ความสามารถหลักของระบบ

### 1.2.1 สำหรับพนักงาน (Employees)

| ความสามารถ | รายละเอียด |
|-----------|-----------|
| **เรียนหลักสูตร** | ลงทะเบียนและเรียนหลักสูตรออนไลน์ได้ตลอด 24 ชั่วโมง |
| **ทำข้อสอบ** | ทำข้อสอบประเมินความรู้หลังเรียน |
| **ดูความก้าวหน้า** | ติดตามความก้าวหน้าการเรียนของตนเอง |
| **รับใบประกาศ** | รับใบประกาศนียบัตรเมื่อเรียนจบหลักสูตร |
| **สะสมเหรียญรางวัล** | สะสม Badges จากกิจกรรมต่างๆ |
| **อ่านบทความ** | เข้าถึงฐานความรู้และบทความต่างๆ |

### 1.2.2 สำหรับผู้สอน/Instructor

| ความสามารถ | รายละเอียด |
|-----------|-----------|
| **สร้างหลักสูตร** | สร้างและจัดการหลักสูตรการเรียน |
| **สร้างข้อสอบ** | สร้างข้อสอบและธนาคารคำถาม |
| **ดูผลการเรียน** | ดูผลการเรียนของผู้เรียนในหลักสูตร |
| **เขียนบทความ** | เขียนและเผยแพร่บทความความรู้ |

### 1.2.3 สำหรับ HR

| ความสามารถ | รายละเอียด |
|-----------|-----------|
| **จัดการผู้สมัคร** | จัดการข้อมูลและการสอบผู้สมัครงาน |
| **สร้างรหัสสอบ** | สร้างรหัสสอบสำหรับผู้สมัคร |
| **ดูผลสอบ** | ดูและวิเคราะห์ผลสอบผู้สมัคร |
| **จัดการผู้ใช้** | เพิ่ม/แก้ไข/ลบผู้ใช้งาน |
| **ออกรายงาน** | ออกรายงานต่างๆ |

### 1.2.4 สำหรับ Admin

| ความสามารถ | รายละเอียด |
|-----------|-----------|
| **จัดการทุกอย่าง** | มีสิทธิ์เต็มในทุกฟังก์ชัน |
| **ตั้งค่าระบบ** | ตั้งค่าทั่วไป, ความปลอดภัย, อีเมล |
| **จัดการองค์กร** | จัดการโครงสร้างองค์กร, แผนก, ตำแหน่ง |
| **ดู Audit Log** | ดูประวัติการเปลี่ยนแปลงระบบ |

# บทที่ 2: ความต้องการของระบบ

## 2.1 ความต้องการของ Browser

### 2.1.1 Browser ที่รองรับ

| Browser | เวอร์ชันขั้นต่ำ | สถานะ | หมายเหตุ |
|---------|:-------------:|:-----:|---------|
| **Google Chrome** | 90+ | แนะนำ | ประสิทธิภาพดีที่สุด |
| **Microsoft Edge** | 90+ | รองรับ | Chromium-based |
| **Mozilla Firefox** | 88+ | รองรับ | - |
| **Safari** | 14+ | รองรับ | macOS/iOS |
| **Opera** | 76+ | รองรับ | Chromium-based |

### 2.1.2 Browser ที่ไม่รองรับ

| Browser | เหตุผล |
|---------|-------|
| **Internet Explorer** | ยุติการสนับสนุนแล้ว |
| **Chrome เวอร์ชันเก่า (<90)** | ไม่รองรับ Feature ใหม่ |
| **Safari เวอร์ชันเก่า (<14)** | ปัญหา JavaScript compatibility |

## 2.2 ความต้องการของอุปกรณ์

### 2.2.1 คอมพิวเตอร์ (Desktop/Laptop)

| รายการ | ขั้นต่ำ | แนะนำ |
|--------|--------|-------|
| **หน้าจอ** | 1024 x 768 px | 1920 x 1080 px |
| **RAM** | 2 GB | 4 GB+ |
| **Internet** | 1 Mbps | 5 Mbps+ |

### 2.2.2 แท็บเล็ต/มือถือ

| รายการ | ขั้นต่ำ |
|--------|--------|
| **iOS** | iOS 14+ |
| **Android** | Android 10+ |
| **หน้าจอ** | 768 px (width) |

## 2.3 การตั้งค่า Browser ที่จำเป็น

### 2.3.1 JavaScript
- **สถานะ:** ต้องเปิดใช้งาน (Required)
- **วิธีตรวจสอบ:** ไปที่ Settings → Privacy and Security → JavaScript → เปิด

### 2.3.2 Cookies
- **สถานะ:** ต้องเปิดใช้งาน (Required)
- **ประเภท:** Session Cookies และ Persistent Cookies
- **วิธีตรวจสอบ:** ไปที่ Settings → Privacy → Cookies → อนุญาต

### 2.3.3 Pop-ups
- **สถานะ:** แนะนำให้อนุญาตสำหรับ domain ของระบบ
- **ใช้สำหรับ:** การพิมพ์ใบประกาศ, รายงาน

## 2.4 การเชื่อมต่อเครือข่าย

### 2.4.1 ความเร็ว Internet ที่แนะนำ

| การใช้งาน | ความเร็วขั้นต่ำ | ความเร็วแนะนำ |
|----------|:-------------:|:------------:|
| **ใช้งานทั่วไป** | 1 Mbps | 5 Mbps |
| **ดูวิดีโอ (720p)** | 3 Mbps | 5 Mbps |
| **ดูวิดีโอ (1080p)** | 5 Mbps | 10 Mbps |
| **ทำข้อสอบ** | 1 Mbps | 3 Mbps |

### 2.4.2 Firewall และ Proxy

หากองค์กรใช้ Firewall หรือ Proxy ต้องอนุญาต:

| Domain/URL | Port | Protocol | การใช้งาน |
|------------|:----:|:--------:|----------|
| `localhost:3000` | 3000 | HTTP | Development |
| `*.youtube.com` | 443 | HTTPS | วิดีโอ YouTube |
| `*.vimeo.com` | 443 | HTTPS | วิดีโอ Vimeo |
| `*.googleapis.com` | 443 | HTTPS | Google Fonts |

# บทที่ 3: บทบาทและสิทธิ์ผู้ใช้

## 3.1 บทบาทในระบบ

ระบบ LearnHub มี 5 บทบาท (Roles) หลัก:

```
                    ┌─────────────────┐
                    │  Administrator  │
                    │   (ผู้ดูแลระบบ)   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
        ┌─────┴─────┐  ┌─────┴─────┐  ┌─────┴─────┐
        │    HR     │  │ Instructor│  │           │
        │ (ฝ่ายบุคคล) │  │  (ผู้สอน)  │  │           │
        └─────┬─────┘  └───────────┘  │           │
              │                        │           │
              │              ┌─────────┴───────────┤
              │              │                     │
        ┌─────┴─────┐  ┌─────┴─────┐         ┌─────┴─────┐
        │ Applicant │  │ Employee  │         │           │
        │ (ผู้สมัคร)  │  │  (พนักงาน) │         │           │
        └───────────┘  └───────────┘         └───────────┘
```

## 3.2 รายละเอียดแต่ละบทบาท

### 3.2.1 Administrator (ผู้ดูแลระบบ)

**คำอธิบาย:** ผู้ที่มีสิทธิ์สูงสุดในการจัดการระบบ

**สิทธิ์การใช้งาน:**

| โมดูล | สิทธิ์ |
|-------|:-----:|
| Dashboard | ดู + ดูทุกคน |
| หลักสูตร | ดู + สร้าง + แก้ไข + ลบ |
| ข้อสอบ | ดู + สร้าง + แก้ไข + ลบ |
| บทความ | ดู + สร้าง + แก้ไข + ลบ |
| ผู้ใช้งาน | ดู + สร้าง + แก้ไข + ลบ |
| ผู้สมัครงาน | ดู + สร้าง + แก้ไข + ลบ |
| รายงาน | ดูทั้งหมด + Export |
| โครงสร้างองค์กร | ดู + แก้ไข |
| ตั้งค่าระบบ | ดู + แก้ไข |
| Audit Log | ดู |

### 3.2.2 HR (ฝ่ายบุคคล)

**คำอธิบาย:** ผู้รับผิดชอบด้านบุคลากรและการสรรหา

**สิทธิ์การใช้งาน:**

| โมดูล | สิทธิ์ |
|-------|:-----:|
| Dashboard | ดู + ดูทุกคน |
| หลักสูตร | ดู + ลงทะเบียน |
| ข้อสอบ | ดู + ทำข้อสอบ |
| บทความ | ดู |
| ผู้ใช้งาน | ดู + สร้าง + แก้ไข |
| ผู้สมัครงาน | ดู + สร้าง + แก้ไข + ลบ |
| รายงาน | ดูทั้งหมด + Export |
| โครงสร้างองค์กร | ดู |
| ตั้งค่าระบบ | ❌ ไม่มีสิทธิ์ |

### 3.2.3 Instructor (ผู้สอน)

**คำอธิบาย:** ผู้สร้างและจัดการเนื้อหาหลักสูตร

**สิทธิ์การใช้งาน:**

| โมดูล | สิทธิ์ |
|-------|:-----:|

### 4.2.3 สิ่งที่เกิดขึ้นเมื่อ Logout

| ลำดับ | การทำงาน | รายละเอียด |
|:-----:|----------|-----------|
| 1 | Session ถูกทำลาย | ลบ Session ทั้งหมดจาก Server |
| 2 | JWT Token ถูกยกเลิก | Token จะไม่สามารถใช้งานได้อีก |
| 3 | Cookies ถูกลบ | ลบ Authentication cookies |
| 4 | Redirect | นำไปหน้า Login |
| 5 | Cache | ล้าง Cache ของผู้ใช้ |

### 4.2.4 การ Logout อัตโนมัติ

ระบบจะ Logout อัตโนมัติเมื่อ:

| สถานการณ์ | เงื่อนไข | การแจ้งเตือน |
|----------|---------|-------------|
| **Session Timeout** | ไม่มีการใช้งานเกิน 24 ชม. | แสดง popup ก่อน 5 นาที |
| **Token หมดอายุ** | JWT Token expire | Redirect ทันที |
| **Admin บังคับ Logout** | Admin กดปุ่ม Force Logout | แสดง popup แจ้ง |
| **เปลี่ยนรหัสผ่าน** | เปลี่ยนรหัสผ่านสำเร็จ | ต้อง Login ใหม่ |

# บทที่ 5: การจัดการรหัสผ่าน

## 5.1 ข้อกำหนดรหัสผ่าน

### 5.1.1 ข้อกำหนดพื้นฐาน

รหัสผ่านต้องมีคุณสมบัติดังนี้:

| ข้อกำหนด | รายละเอียด | ตัวอย่างที่ถูก | ตัวอย่างที่ผิด |
|---------|-----------|:-------------:|:--------------:|
| **ความยาว** | อย่างน้อย 8 ตัวอักษร | `MyPass123` | `Pass1` |
| **ตัวพิมพ์ใหญ่** | อย่างน้อย 1 ตัว (A-Z) | `MyPass123` | `mypass123` |
| **ตัวพิมพ์เล็ก** | อย่างน้อย 1 ตัว (a-z) | `MyPass123` | `MYPASS123` |
| **ตัวเลข** | อย่างน้อย 1 ตัว (0-9) | `MyPass123` | `MyPassword` |

### 5.1.2 ข้อกำหนดเพิ่มเติม (ตั้งค่าได้)

| ข้อกำหนด | ค่าเริ่มต้น | รายละเอียด |
|---------|:---------:|-----------|
| **ตัวอักษรพิเศษ** | ❌ ปิด | อักขระ !@#$%^&*()_+-=[]{}|;:,.<>? |
| **ความยาวขั้นต่ำ** | 8 | ตั้งค่าได้ 6-20 |
| **ห้ามใช้ซ้ำ** | ❌ ปิด | ห้ามใช้รหัสผ่านเดิม X ครั้งหลังสุด |

### 5.3.3 ขั้นตอนการรีเซ็ตรหัสผ่าน

1. **คลิกลิงก์ในอีเมล** → ระบบจะนำไปหน้ารีเซ็ตรหัสผ่าน
2. **กรอกรหัสผ่านใหม่** → ตามข้อกำหนดรหัสผ่าน
3. **กรอกรหัสผ่านใหม่อีกครั้ง** → ต้องตรงกัน
4. **ตรวจสอบ Checklist** → ต้องครบทุกข้อ (เครื่องหมาย ✅ สีเขียว)
5. **คลิกปุ่ม "รีเซ็ตรหัสผ่าน"**
6. **ระบบนำไปหน้า Login** → พร้อมข้อความ "รีเซ็ตรหัสผ่านสำเร็จ"

### 5.3.4 ข้อผิดพลาดที่อาจเกิดขึ้น

| ข้อผิดพลาด | สาเหตุ | วิธีแก้ไข |
|-----------|--------|----------|
| "ลิงก์หมดอายุหรือไม่ถูกต้อง" | ลิงก์เกิน 1 ชม. หรือใช้แล้ว | ขอลิงก์ใหม่ที่หน้าลืมรหัสผ่าน |
| "รหัสผ่านไม่ตรงตามข้อกำหนด" | ไม่ครบตาม Checklist | แก้ไขรหัสผ่านตามข้อกำหนด |
| "รหัสผ่านทั้งสองช่องไม่ตรงกัน" | พิมพ์ไม่ตรงกัน | กรอกใหม่ให้ตรงกัน |

## 5.4 การเปลี่ยนรหัสผ่าน (Change Password)

### 5.4.1 การเข้าถึง

**วิธีที่ 1: ผ่าน Profile**

2. เลือก **"โปรไฟล์"**

**วิธีที่ 2: ผ่าน URL โดยตรง**
```

### 5.4.3 ขั้นตอนการเปลี่ยนรหัสผ่าน

1. **กรอกรหัสผ่านปัจจุบัน** → ต้องถูกต้อง
2. **กรอกรหัสผ่านใหม่** → ตามข้อกำหนดรหัสผ่าน
3. **กรอกรหัสผ่านใหม่อีกครั้ง** → ต้องตรงกัน
4. **ตรวจสอบ Checklist** → ต้องครบทุกข้อ
5. **คลิกปุ่ม "บันทึก"**
6. **ระบบแสดงข้อความสำเร็จ** และอาจต้อง Login ใหม่

## 5.5 การบังคับเปลี่ยนรหัสผ่าน (Force Change Password)

## 7.2 การค้นหาและกรองหลักสูตร

### 7.2.1 การค้นหา

**ขั้นตอน:**
1. พิมพ์คำค้นหาในช่อง Search
2. กด Enter หรือคลิกไอคอน 🔍
3. ระบบจะแสดงผลลัพธ์ที่ตรงกับคำค้นหา

**ค้นหาได้จาก:**
- ชื่อหลักสูตร
- คำอธิบาย
- รหัสหลักสูตร

### 7.2.2 ตัวกรอง (Filters)

| ตัวกรอง | ตัวเลือก | ค่าเริ่มต้น |
|---------|---------|:---------:|
| **หมวดหมู่** | ทั้งหมด, ความปลอดภัย, IT, HR, ... | ทั้งหมด |
| **ระดับความยาก** | ทั้งหมด, ง่าย, ปานกลาง, ยาก | ทั้งหมด |
| **สถานะ** | ทั้งหมด, ยังไม่ลงทะเบียน, กำลังเรียน, เรียนจบ | ทั้งหมด |
| **ระยะเวลา** | ทั้งหมด, < 1 ชม., 1-5 ชม., > 5 ชม. | ทั้งหมด |

### 7.2.3 การเรียงลำดับ

| ตัวเลือก | รายละเอียด |
|---------|-----------|
| **ล่าสุด** | เรียงตามวันที่สร้าง (ใหม่สุดก่อน) |

## ง.1 คำถามทั่วไป

**Q: ฉันสามารถเรียนหลักสูตรได้ตลอด 24 ชั่วโมงหรือไม่?**
A: ได้ ระบบเปิดให้บริการตลอด 24 ชั่วโมง ทุกวัน

**Q: ฉันสามารถเรียนบนมือถือได้หรือไม่?**
A: ได้ ระบบรองรับ Responsive Design สามารถใช้งานบน Mobile Browser ได้

**Q: หากฉันไม่สามารถเรียนจบหลักสูตรในครั้งเดียว จะเกิดอะไรขึ้น?**
A: ระบบจะบันทึกความก้าวหน้าอัตโนมัติ คุณสามารถกลับมาเรียนต่อได้ทุกเมื่อ

## ง.2 คำถามเกี่ยวกับข้อสอบ

**Q: ถ้าหมดเวลาสอบแล้วยังไม่ได้ส่งจะเกิดอะไรขึ้น?**
A: ระบบจะส่งคำตอบที่บันทึกไว้ให้อัตโนมัติเมื่อหมดเวลา

**Q: ฉันสามารถสอบใหม่ได้หรือไม่หากสอบไม่ผ่าน?**
A: ได้ ขึ้นอยู่กับจำนวนครั้งที่กำหนดในแต่ละข้อสอบ

**Q: ฉันสามารถกลับไปแก้ไขคำตอบก่อนส่งได้หรือไม่?**
A: ได้ คุณสามารถไปกลับระหว่างข้อได้จนกว่าจะกดส่ง

## ง.3 คำถามเกี่ยวกับใบประกาศ

**Q: ฉันจะได้รับใบประกาศเมื่อไหร่?**
A: ทันทีที่เรียนจบหลักสูตรและผ่านเกณฑ์ (ถ้าหลักสูตรนั้นมีใบประกาศ)

**Q: ฉันสามารถดาวน์โหลดใบประกาศได้ที่ไหน?**
A: ไปที่ Profile → ใบประกาศนียบัตร หรือในหน้า "หลักสูตรของฉัน"
